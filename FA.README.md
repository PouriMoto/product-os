🎯 نمای کلی

Product OS v1 یک چارچوب جامع و پایگاه دانش برای ساخت محصولات نرم‌افزاری با استفاده از Vibe Coding است — رویکردی توسعه‌ای که برای همکاری با دستیارهای هوش مصنوعی مانند ChatGPT، Claude و ابزارهای مشابه بهینه‌سازی شده است.

این مخزن چه چیزی ارائه می‌دهد

ویژگی مزیت
📄 مستندسازی استاندارد کار با هر دستیار هوش مصنوعی
✅ چک‌لیست‌های عملیاتی راهنمای گام‌به‌گام برای هر مرحله
🎯 کتابخانه پرامپت تعاملات یکپارچه با هوش مصنوعی
📝 ثبت تصمیم‌ها جلوگیری از بحث‌های تکراری
📋 پلی‌بوک‌ها راه‌اندازی سریع محصولات

Vibe Coding چیست؟

Vibe Coding روشی است که در آن:

```
🧠 لایه‌ای فکر کن  →  درک کن → تصمیم بگیر → اجرا کن
📦 اتمیک کار کن   →  هر فایل = یک مفهوم
🤖 هوش مصنوعی هم‌برنامه‌نویس → پرامپت‌های شفاف، خروجی‌های یکسان
📈 افزایشی بساز → توسعه اسپرینت‌محور با انتشارهای پایدار
📚 همه‌چیز را مستند کن → هر تصمیم یک جایگاه دارد
```

ساختار فایل

بیشتر فایل‌ها از این ساختار پیروی می‌کنند (برخی بخش‌ها ممکن است حذف شوند):

بخش هدف

چرا چرا این مفهوم مهم است
چه زمانی چه زمانی از این فایل استفاده کنیم
ورودی‌ها قبل از شروع به چه چیزی نیاز دارید
خروجی‌ها پس از اتمام چه چیزی خواهید داشت
چک‌لیست اقدامات گام‌به‌گام
بهترین روش‌ها رویکردهای توصیه‌شده
اشتباهات رایج چه چیزهایی را باید اجتناب کنید
ابزارها ابزارها و سرویس‌های توصیه‌شده
پرامپت پرامپت هوش مصنوعی (فقط در فایل‌های PRM)
فایل‌های مرتبط ارجاعات متقابل

ساختار سه‌لایه

هر فایل از الگوی درک کن → تصمیم بگیر → اجرا کن پیروی می‌کند:

```
۱️⃣ درک کن    →  این مفهوم چیست؟
۲️⃣ تصمیم بگیر →  چه تصمیمی باید بگیرم؟
۳️⃣ اجرا کن   →  دقیقاً چه کاری انجام دهم؟
```

📝 سیستم ثبت تصمیم‌ها

🎯 هدف

سیستمی سبک برای ثبت تصمیم‌ها و ایده‌ها بدون ایجاد وقفه در کار. هر ایده‌ای به ثبت تصمیم‌ها می‌رود، نه به توسعه فعال.

نحوه کار

مرحله اقدام
۱ اول ثبت کن — هر ایده‌ای → فایل تصمیم ایجاد کن
۲ اقدام نکن — هرگز در اسپرینت جاری پیاده‌سازی نکن
۳ بعداً بازبینی کن — در برنامه‌ریزی اسپرینت ارزیابی کن
۴ ساده نگه دار — حداکثر ۲۰۰ کلمه برای هر تصمیم

ساختار فایل

```markdown
# تصمیم: [عنوان]

**شناسه تصمیم:** SPR-DEC-001
**تاریخ:** ۱۴۰۴-۱۰-۱۹
**وضعیت:** پیشنهادی | تأییدشده | ردشده | به‌تعویق‌افتاده
**فایل‌های مرتبط:** [FD-AUTH-001]

---
## زمینه
چه چیزی باعث این تصمیم شد؟

## گزینه‌های بررسی‌شده
۱. گزینه الف - مزایا/معایب
۲. گزینه ب - مزایا/معایب

## تصمیم
چه چیزی انتخاب کردیم و چرا.

## تأثیر
چه چیزی تغییر می‌کند.

## اقدام
- [ ] چه کاری باید الان انجام داد
- [ ] چه کاری باید به تعویق افتاد

## برای آینده
چگونه بر نسخه ۲ یا اسپرینت‌های بعدی تأثیر می‌گذارد.
```

چرخه حیات تصمیم

```
💡 ایده → 📝 ثبت → 🔍 بازبینی → ✅ تأیید → 🚀 پیاده‌سازی
                                ↓
                            ❌ رد
                                ↓
                            ⏸️ به‌تعویق (v2)
```

وضعیت‌ها

وضعیت معنی اقدام بعدی
پیشنهادی ایده جدید، نیاز به بررسی بررسی در برنامه‌ریزی اسپرینت بعدی
تأییدشده آماده پیاده‌سازی اضافه به بکلاگ اسپرینت بعدی
ردشده پیاده‌سازی نمی‌شود دلیل را مستند کن، ببند
به‌تعویق‌افتاده ایده خوب، اما برای v2 انتقال به بکلاگ v2

🚀 شروع به کار

استفاده از این مخزن

۱. با Sprint 00 شروع کنید — استانداردها و پروتکل‌ها را بخوانید
۲. ترتیب اسپرینت‌ها را رعایت کنید — هر اسپرینت بر اساس قبلی ساخته می‌شود
۳. از شناسه فایل‌ها استفاده کنید — با شناسه درخواست دهید (مثلاً "FD-AUTH-001")
۴. اتمیک کار کنید — هر فایل = یک مفهوم
۵. زنده نگه دارید — بر اساس تجربه پروژه‌های واقعی به‌روزرسانی کنید

کار با دستیارهای هوش مصنوعی

به سادگی یک فایل را با شناسه آن درخواست دهید:

```
"FD-AUTH-001 را نشان بده"
"BP-ARCH-001 را به نسخه ۱.۱ به‌روزرسانی کن"
"یک فایل جدید FD-NEW-001 با [توضیحات] ایجاد کن"
```

هوش مصنوعی دقیقاً محتوای آن فایل را طبق همه استانداردها پاسخ می‌دهد.

ایجاد فایل‌های جدید

مرحله اقدام
۱ دسته مناسب را انتخاب کنید (PRN/CAN/BP/FD/PRM/TMP/QA/EX/PLY)
۲ کد موضوع را انتخاب کنید
۳ شماره بعدی موجود را تعیین کنید
۴ از ساختار استاندارد فایل استفاده کنید
۵ ارجاعات متقابل اضافه کنید
۶ وضعیت را "پیش‌نویس" قرار دهید
۷ در فهرست مطالب قرار دهید

جلوگیری از گسترش دامنه

· ❌ اسپرینت‌های تکمیل‌شده را تغییر ندهید
· 📝 ایده‌های جدید را در ثبت تصمیم‌ها وارد کنید
· 🚀 تغییرات اساسی را برای v2 نگه دارید
· 🎯 هر اسپرینت را متمرکز نگه دارید

دستورالعمل‌ها

# دستورالعمل
۱ از همه استانداردهای مستندسازی پیروی کنید
۲ فایل‌های اتمیک را حفظ کنید (هر فایل = یک موضوع)
۳ محتوای عملیاتی نگه دارید (درک کن → تصمیم بگیر → اجرا کن)
۴ ارجاعات متقابل اضافه کنید، نه محتوای تکراری
۵ هنگام تغییر، نسخه و وضعیت را به‌روزرسانی کنید
۶ هرگز اسپرینت‌های فریز شده را تغییر ندهید

فرآیند درخواست تغییر

۱. محتوای موجود را بررسی کنید تا از تکرار جلوگیری شود
۲. اطمینان حاصل کنید که فایل شما از همه استانداردها پیروی می‌کند
۳. بخش فایل‌های مرتبط را اضافه کنید
۴. فهرست مطالب مخزن را به‌روزرسانی کنید
۵. درخواست بازبینی از نگهدارندگان دهید
